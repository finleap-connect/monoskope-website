Meet Monoskope

Monoskope (short m8 spelled "mate") implements the management and operation of tenants, users and their roles in a Kubernetes multi-cluster environment.

It fulfills the needs of operators on the clusters as well as the needs of developers using the cloud infrastructure provided by the operators.

Learn more:

  • Monoskope Architecture

    • Powerful CLI It gives your developers a front-end to your internal platform for running Kubernetes.

    Self-Service It allows developers to request what they need to be productive and run application workloads while avoiding cognitive overload.

    Modularity It is designed to be extendable to allow for integration with your existing components.

    Practical Features

    Secure by default

    zero trust, zero touch credentials Flexible, enterprise-ready authentication through Dex

    Auditable by design

    For ANY and ALL changes to the configuration a record of who changes what and when is kept. The audit record is a first class feature and not some logging afterthought.

    Easy installation

    through Helm charts to any vanilla Kubernetes cluster
